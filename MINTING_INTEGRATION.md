# AI NFT Minting Integration Guide

This guide walks through the complete AI-to-NFT minting flow that has been implemented.

## Architecture Overview

The minting integration consists of three main components:

1. **Image Generation** (existing) - AIGeneratorChat with Cloudflare Workers AI
2. **IPFS Upload** (new) - `useIPFSUpload` hook for persistent storage
3. **NFT Minting** (new) - `useNFTMint` hook for on-chain transactions

## Component Overview

### `hooks/useIPFSUpload.ts`
Uploads AI-generated images to IPFS via Pinata or NFT.storage

**Usage:**
```tsx
const { upload, isLoading, ipfsUrl, error } = useIPFSUpload();

// Upload an image
const result = await upload(imageDataUrl, 'my-nft.png');
if (result.success) {
  console.log('IPFS URL:', result.ipfsUrl);
}
```

**Providers:**
- **NFT.storage** (default): Free, no rate limits, requires API key
- **Pinata**: Paid tier, pinning service with dashboard

### `hooks/useNFTMint.ts`
Handles metadata preparation and contract interaction for minting

**Usage:**
```tsx
const { mint, isLoading, isSuccess, error } = useNFTMint(
  contractAddress,
  contractABI
);

// Mint an NFT
const result = await mint({
  imageUri: 'ipfs://QmXxxx...',
  prompt: 'A futuristic robot warrior',
  name: 'AI Robot #1',
  description: 'Generated by Mintly AI'
});
```

### `components/modules/AIGenerator/AIGeneratorChat.tsx`
Chat interface with integrated mint button below each generated image

**New Features:**
- "Mint NFT" button under each generated variation
- Automatic IPFS upload on mint click
- Transaction state UI (uploading → minting → success/error)  
- Wallet connection check before minting
- Error messages for failed transactions

## Setup Instructions

### Step 1: Choose IPFS Provider

#### Option A: NFT.storage (Recommended for beginners)

1. Go to [nft.storage](https://nft.storage)
2. Sign up with GitHub
3. Create API key
4. Add to `.env.local`:
```
NEXT_PUBLIC_IPFS_PROVIDER=nft-storage
NEXT_PUBLIC_NFT_STORAGE_KEY=eyJhbGc...
```

#### Option B: Pinata

1. Go to [pinata.cloud](https://www.pinata.cloud)
2. Create account and generate API keys
3. Add to `.env.local`:
```
NEXT_PUBLIC_IPFS_PROVIDER=pinata
NEXT_PUBLIC_PINATA_API_KEY=xxx
PINATA_API_SECRET=yyy
```

### Step 2: Set Up Smart Contract

You need:
1. A deployed ERC721 or ERC1155 contract with a `mint` function
2. Contract address (e.g., `0x123abc...`)
3. Contract ABI (JSON representation)
4. Deploy on testnet first (Sepolia recommended)

Example contract structure:
```solidity
contract AIGeneratedNFT is ERC721 {
  function mint(address to, string memory tokenURI) public returns (uint256) {
    // Implementation
  }
}
```

#### Add to `.env.local`:
```
NEXT_PUBLIC_NFT_CONTRACT_ADDRESS=0x...
NEXT_PUBLIC_NFT_CONTRACT_CHAIN_ID=11155111  # Sepolia
```

### Step 3: Configure Contract ABI

Add your contract ABI in code. Update `hooks/useNFTMint.ts`:

```tsx
// Example: Standard ERC721 mint ABI
const contractABI = [
  {
    "inputs": [
      { "internalType": "address", "name": "to", "type": "address" },
      { "internalType": "string", "name": "tokenURI", "type": "string" }
    ],
    "name": "mint",
    "outputs": [{ "internalType": "uint256", "name": "", "type": "uint256" }],
    "stateMutability": "nonpayable",
    "type": "function"
  }
];
```

### Step 4: Update AIGeneratorChat Component

Currently, `handleMintImage` in AIGeneratorChat logs the data but doesn't actually call the contract. To enable full minting:

1. Pass contract configuration to AIGeneratorChat:
```tsx
<AIGeneratorChat
  contractAddress={process.env.NEXT_PUBLIC_NFT_CONTRACT_ADDRESS}
  contractABI={YOUR_CONTRACT_ABI}
/>
```

2. Update `handleMintImage` to call `mint`:
```tsx
const result = await mint({
  imageUri: uploadResult.ipfsUrl,
  prompt: generatedImages.prompt,
  name: `AI NFT #${imageIndex + 1}`,
});
```

## Error Handling

The implementation includes comprehensive error handling:

- **Wallet not connected** - Shows "Connect Wallet" state
- **IPFS upload fails** - Displays error message, retryable
- **Contract interaction fails** - Shows specific error from contract
- **Metadata composing fails** - Validates before upload

## Troubleshooting

### "Connect your wallet first"
- MetaMask not connected
- Solution: Click wallet button in UIcard

### "Failed to upload to IPFS"
- Invalid API key
- Network error
- Solution: Check env vars and network connection

### "Minting failed: Contract call failed"
- Contract address wrong
- ABI mismatch
- Not enough gas
- Solution: Verify contract is deployed and test on Sepolia

### "ipfsUrl is null"
- Upload didn't complete
- Solution: Check IPFS provider status

## Advanced: Custom Metadata

Modify metadata in `useNFTMint.ts`:

```tsx
const metadata = {
  name: `${params.name}`,
  description: `${params.description}`,
  image: params.imageUri,
  prompt: params.prompt,
  attributes: [
    { trait_type: 'Generator', value: 'Mintly AI' },
    { trait_type: 'Model', value: 'Cloudflare Workers AI' },
    { trait_type: 'Prompt Length', value: params.prompt.length },
  ],
};
```

## Security Notes

1. **API Keys**: Store in `.env.local` (never commit)
2. **Contract Validation**: Always verify contract address on blockchain
3. **Gas Limits**: Set appropriate gas limits for contract calls
4. **IPFS Pinning**: Use service with retention guarantees

## Performance Optimization

- Lazy load mint hook only when visible
- Debounce rapid mint clicks
- Cache contract ABI
- Preload IPFS gateway URLs
